Avant on avait une équipe de développement qui bossait sur un projet et une fois qu’ils ont fini ils l'envoyaient à l'équipe opérationnelle qui s'occupait des serveurs.

Opérationnelle : déployer l’application sur le serveur. L'application ne marche pas => ils contactent les développeurs

Développeurs : le code marche bien chez nous, regardez vos serveurs.

Opérationnelle : les serveurs sont bien chez nous, c’est votre code qui marche pas.

=> ni l’un ni l’autre comprend le travail de l’autre

Solution : 

DevOps : développeur opérationnelle

3 étapes que le DevOps fait :

  • Création de l’environnement
  • Test
  • Deploy
